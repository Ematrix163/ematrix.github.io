CSS 布局详解

常用的布局方法:

  • table 表格布局

    早期浏览器对于表格的解析不是流式的,如果代码很长,浏览器会将表格拉完之后再解析

  • float 浮动 + margin

  • inline-block 布局

  • flexbox 布局

表格布局

Codepen Demo: https://codepen.io/ematrix163/pen/qMjdbX?editors=1100

第一种方式是用自带的table标签等,第二种方式是用display:table模拟table. 这种方式可以方便做到垂直居中.

Flexbox 布局

  • 弹性盒子

  • 盒子本来就是并列的

  • 制定宽度即可

目前为止, 依然没有被大规模的使用. 主要原因还是兼容性上的问题, 低版本的IE是完全不支持的. Flex 目前有3个版本, 不同浏览器兼容情况不一样.

Float布局

特点

  • 元素浮动
  • 脱离文档流
  • 但不脱离文本流(影响文字布局,使其它元素的文字绕开)

影响

对自身的影响

  • 形成”块” (BFC) (比如可以让inline元素设置高度)

  • 位置先尽量靠上,然后尽量靠左(float:left)

对兄弟的影响

  • 上面帖非float元素
  • 旁边帖float元素
  • 不影响其它块级元素位置
  • 影响其它块级元素内部的文本

对父级的影响

  • 从布局上”消失”,导致父级元素塌陷

    解决方案:

    1. 对父元素设置overflow:auto,自动监测子元素有没有超出

Inline-block布局

  • 像文本一样排block元素

  • 没有清除浮动的问题

  • 需要处理间隙

    inline-block元素之间有间隙, 因为类似于文字, 文字之间有间隙

    消灭间隙的办法:

    1. 父元素的字体大小设置为0 font-size: 0, 在里面的元素中再把文字大小设置回来

    2. 消除html里面两个div之间的空格, 或者在两个元素之间加注释符号<!--!-->

  • 适合定宽的

布局相关的面试题

  1. 实现两栏(三栏)布局的方法

    • 表格布局

    • float + margin 布局 (主流方式, 兼容性好)

    • inline-block 布局

    • flexbox布局 （兼容性不佳）

  2. position: absolute/fixed 有什么区别？

    都是绝对定位，但是参照物不同：

    • absolute是相对最近的absolute/fixed定位
    • 后者相对于屏幕 (viewport) 定位， 在移动端表现不佳

  3. display: inline-block 为什么有间隙？

    原因： 字符间距

    解决：

    • 消灭字符 font-size:0
    • 消灭间距

  4. 为什么以及如何清除浮动？

    原因：浮动的元素不会占据父元素的布局空间，父元素布局的时候不会去管浮动元素，浮动元素因此可能会超出父元素， 从而对其它元素产生影响。因此作为父元素必须要清楚浮动，保证无毒无害，不会影响其它元素。

    方法：

    (1） 让盒子负责自己的布局

    overflow:hidden(auto)

    (2) 添加一个伪元素：

    ::after{clear:both}

  5. 如何适配移动端页面？

    • viewport
    • rem/viewport/media query
    • 设计上： 隐藏 折行 自适应
